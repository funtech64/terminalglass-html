Model lanes

Models for Terminal.Glass

Terminal.Glass organizes models into useful lanes: NoCloudGPT local/private models for customer-controlled infrastructure and Jet Agent cloud-model lanes for speed, coding, research, drafting, and model variety.

NoCloudGPT local/private model lane

Models run on customer-owned hardware, on-prem boxes, or customer-controlled servers. This is the local/private lane when data locality and infrastructure control matter.

Jet Agent cloud-model lane

Curated cloud-model workflows for stronger answers and faster reasoning without buying a large GPU server first. Jet Agents are not separate licenses; they live inside Glass License package capacity.

Glass Agent action lane

OpenClaw-based workers that can act inside approved customer environments. Glass Agents are the action-capable lane for future Alchemist coordination.

Alchemist-ready workflows

Alchemist Conductor will coordinate Glass Agents and Jet Agents into business workflows with owner approval, logs, routing, and memory.
