Paths

Self-Hosted or Managed

Two operational models. Pick the one that matches how your team runs infrastructure.

Self-Hosted

Customer-owned Linux server — on-premise or private datacenter. You control hardware, network, and patching. Best for maximum operational control.

  • You provision and own the server
  • Inference stays on your infrastructure (local models)
  • Terminal.Glass installs and configures the stack

Managed Hosting

Server provisioned and configured for you. Terminal.Glass stack installed and prepared. A simpler starting path when you want deployment handled.

  • We coordinate server provisioning with your team
  • Terminal.Glass software setup included
  • Infrastructure ownership and billing explained before launch

Sizing

Hardware and Model Sizing

Model capability depends on server resources. We recommend a hosting size with every order.

Local and self-hosted models

RAM and GPU requirements vary by model family. Small models may run on CPU-only hosts with 8–16 GB RAM. Larger reasoning models and GPU workloads need appropriately sized hardware. Each model page in the catalog includes a recommended hosting size.

Cloud-accelerated models

Some Terminal.Glass deployments use Ollama Cloud-backed models. Your host runs Open WebUI and the stack on a modest Linux server — typically 4–8 GB RAM, no GPU required on the host. Inference is handled by Ollama Cloud; prompts leave your network per Ollama's terms. This is cloud-accelerated model inference with a small private Linux host — not a fully offline deployment.
