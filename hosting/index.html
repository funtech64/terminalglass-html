Choices

Four deployment paths

Choose where your private AI system lives. Every path delivers a familiar ChatGPT-style interface with access to 100+ models.

Install on my server

Hosted privately on one Linux server you own — office hardware, on-premise systems, clinics, shops, or other machines under your control.

  • You own the server
  • Models can stay on your infrastructure
  • Best when local control matters most

Launch in my cloud

A private AI server in a cloud account you control — such as AWS or a private VPS. You own the account; Terminal.Glass handles deployment.

  • Good when local hardware is not ready
  • Customer-owned cloud deployment
  • Start modest and scale as needed

Add agent workflows

Action-capable workflows that run inside approved environments on your private AI system — for teams that need more than chat.

  • Approved environment boundary
  • Extends your private AI server
  • Built with OpenClaw

Add cloud model access

Access stronger models through cloud inference while your private AI server handles the chat interface, user accounts, and document chat.

  • More model power without a large local GPU
  • Third-party usage billed separately
  • Your team still uses one familiar interface

Ownership

Your server. Your data. Your control.

The standard path is customer-owned: you install or launch your private AI server in your own environment. Terminal.Glass does not need access to your server by default.

White-glove setup can be offered as a support option, but customer ownership remains the default trust model. Infrastructure and third-party model usage are billed separately unless included in a written managed-hosting agreement.
