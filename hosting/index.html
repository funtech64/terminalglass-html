Choices

Four clean paths

The sales app should not need to log into a customer server by default. It should create an easy install, launch, or activation path.

Install on my server - NoCloudGPT

Local/private GPT host running on customer-owned hardware, office Linux servers, homelabs, clinics, shops, churches, or other customer-controlled systems.

  • Customer owns the server
  • Local models can stay on customer infrastructure
  • Best when local control matters

Launch in my cloud - YourCloudGPT

Customer-owned cloud deployment, such as AWS-hosted Terminal.Glass. The customer owns the cloud account while Terminal.Glass provides the launch and activation path.

  • Good when local hardware is not ready
  • Supports customer-owned AWS-style deployment
  • Can start with modest servers and Jet Agent lanes

Add Glass Agents

OpenClaw-based action workers that act inside approved customer environments. This is the local/action lane for future workflow automation.

  • OpenClaw-powered workflows
  • Approved environment boundary
  • Future Alchemist coordination target

Add Jet Agents

Curated cloud-model lanes for speed, coding, research, drafting, and model variety. Jet Agents are not a separate license type.

  • Useful when GPU hardware is too expensive
  • Cloud model power without buying a big local GPU server first
  • Available through Glass License package capacity
